# 专注页右侧今日任务卡设计说明

## Summary

将专注页右侧“今日任务”区域定义为轻量任务预览卡，而不是任务管理页的缩略版。它的职责是服务专注主流程：展示当前任务上下文、预览今日任务、支持快速切换当前任务、提供新建任务和进入完整任务页的入口。

该模块不承担复杂筛选、批量管理、长表单编辑等任务管理能力。这些能力继续留在 `/tasks` 页面和 `TaskEditorDrawer` 中。

## Goals

- 让用户在 `/focus` 页面内完成高频任务操作，不必频繁跳转 `/tasks`
- 保持右侧栏信息密度适中，不抢占计时器主视觉
- 强化“任务服务专注”的产品定位
- 复用现有 `TaskEditorDrawer`，避免新建/编辑表单重复实现

## Non-Goals

- 不在该卡片内实现复杂筛选、排序、批量操作
- 不展示任务描述、优先级等低频字段
- 不把任务页整套管理能力迁入专注页

## User Experience

### 信息结构

卡片固定分为三段：

1. 头部
   - 标题：`今日任务`
   - 右上角主动作：`+ 添加任务`

2. 列表区
   - 固定展示前 4 条任务
   - 当前专注任务优先置顶
   - 其余任务按未完成优先，再按最近更新时间排序

3. 底部
   - 次动作：`查看全部任务 ->`

### 单条任务内容

每条任务行展示：

- 完成勾选
- 任务标题（单行省略）
- `预计 N` 胶囊
- `当前专注` 状态胶囊（仅当前任务显示）
- `...` 更多操作按钮

### 单条任务交互

- 点击勾选：完成 / 取消完成
- 点击任务主体：设为当前任务
- 点击 `...`：弹出任务操作菜单
  - 设为当前任务
  - 开始专注
  - 编辑
  - 删除

### 空状态

空状态不只展示插画和文案，增加主按钮：

- 标题：`还没有任务`
- 次文案：`先添加一个小目标，再开始一轮更稳`
- 主按钮：`添加任务`

空状态主按钮直接打开 `TaskEditorDrawer`。

## Data Flow

### 数据来源

- 任务列表：`useTaskStore().tasks`
- 当前任务：`useTimerStore().currentTaskId`

### 排序逻辑

1. 当前任务排第一
2. 未完成任务排在已完成任务之前
3. 同状态下按 `updatedAt` 倒序
4. 最终截取前 4 条

### 行为联动

- `+ 添加任务` -> 打开 `TaskEditorDrawer`
- `查看全部任务` -> 跳转 `/tasks`
- `编辑` -> 打开 `TaskEditorDrawer` 并注入当前任务数据
- `开始专注` -> 设为当前任务 -> 跳转 `/focus` -> 触发开始专注

## Component Architecture

按现有代码结构拆分为：

- `FocusTaskCard`
  - 右侧整张任务卡容器
- `FocusTaskList`
  - 负责排序、截断、空状态切换
- `FocusTaskListItem`
  - 单条任务行展示
- `FocusTaskActionsMenu`
  - 更多操作菜单
- `FocusTaskEmptyState`
  - 空状态视图

容器编排放在 `FocusWorkspace` 或该页对应容器组件中，展示组件不直接耦合路由和复杂业务逻辑。

## Visual Guidelines

- 任务卡维持现有奶油白背景与浅描边
- 行高控制在 `52px ~ 60px`
- 不显示描述和优先级标签，避免任务卡过重
- `预计 N` 与 `当前专注` 使用轻胶囊样式
- `...` 为 icon-only 按钮，需补 `aria-label`

## Error Handling

- 任务为空时，展示空状态，不显示空列表
- 当前任务不存在于任务列表时，按普通排序展示
- 任务标题过长时单行省略

## Testing

需要覆盖：

- 有任务时仅显示前 4 条
- 当前任务优先置顶
- 空状态展示 `添加任务`
- 点击 `+ 添加任务` 打开 `TaskEditorDrawer`
- 点击 `查看全部任务` 跳转 `/tasks`
- 点击 `开始专注` 会设为当前任务并触发主流程

## Implementation Order

1. 新增 `FocusTaskCard / FocusTaskList / FocusTaskListItem / FocusTaskActionsMenu / FocusTaskEmptyState`
2. 将专注页现有右侧任务区替换为新组件
3. 复用 `TaskEditorDrawer` 接入新建/编辑
4. 接入更多操作菜单
5. 补测试
