# 专注页今日任务交互设计说明

## Summary

将专注页右侧“今日任务”模块定义为 `Today List + Focus Queue`，服务番茄执行流，而不是通用任务列表的缩略版。它的职责是：

- 告诉用户今天准备做什么
- 明确当前最该做哪一项
- 提供一键开始专注的主入口
- 在番茄完成后自然衔接到下一项

该模块优先吸收 `Pomofocus / Focus To-Do` 的轻任务执行模型，并借鉴 `Todoist / TickTick / Sunsama` 在 Today 视图、优先级、专注模式上的成熟交互。

## Goals

- 建立真正的“今日任务”概念，而不是简单展示全部未完成任务
- 让 `开始专注` 成为列表项唯一明确的主动作
- 在列表项中显示番茄执行上下文，降低用户决策成本
- 让当前任务、专注中任务、已完成任务的状态一眼可辨
- 保持右侧卡片轻量，不侵占计时器主视觉

## Non-Goals

- 不在专注页实现完整任务管理页能力
- 不在第一版引入复杂筛选、批量操作、长表单编辑
- 不在第一版引入多级子任务编排和复杂日程排程
- 不把该卡片做成重型 Todo / Project 面板

## Current Gaps

基于当前实现，存在以下产品和交互问题：

1. `今日任务` 语义不成立  
   当前数据实际来自全部未完成任务过滤，而非“今天准备执行的任务”集合。

2. 主动作不够明确  
   列表项整行点击会设为当前任务，同时右侧还有一个很小的 `Play` 图标用于开始专注，动作分层不清。

3. 执行上下文缺失  
   数据模型已有 `estimatedPomodoros` 与 `completedPomodoros`，但列表项未展示番茄进度，用户无法判断任务体量和推进情况。

4. 菜单动作过薄  
   当前更多菜单只有编辑和删除，不足以支撑 Today List 的实际使用场景。

## Product Positioning

专注页右侧模块应采用“双层任务模型”：

- `任务库`
  - 全部任务
  - 在 `/tasks` 页面集中管理

- `今日任务`
  - 今天准备执行的任务集合
  - 在 `/focus` 页面用于执行和切换专注

如果暂时不引入新的数据字段，第一阶段可以先将当前模块定义为“专注任务队列预览”；但正式方向应落到“今日任务”。

## User Experience

### 卡片结构

卡片固定分为三段：

1. `Header`
   - 标题：`今日任务`
   - 摘要：`已完成 x / y`
   - 主按钮：`添加任务`

2. `List`
   - 默认展示 4 条
   - 超出后展示 `查看全部任务`
   - 列表默认按建议执行顺序排序

3. `Footer / Inline CTA`
   - 有当前任务且未开始时：`开始当前任务`
   - 今日全部完成时：`今日任务已完成`

### 列表项信息结构

每个任务项固定为四区：

1. 完成按钮
2. 标题与次信息
3. 状态标签
4. 主动作与更多菜单

推荐线框：

```text
[完成圆框]  任务标题
           1/3 个番茄 · 高优先级
           [当前任务 / 专注中]
                         [开始专注] [···]
```

### 次信息规则

列表项至少显示：

- 番茄进度：`1/3 个番茄`
- 优先级：仅对 `important / urgent` 进行提示

展示优先级的规则：

- `urgent`：红点 + `高优先级`
- `important`：橙点 + `重要`
- `normal`：不额外展示文字，避免噪音

### 状态标签规则

同一时刻只显示一个标签：

- `当前任务`
- `专注中`
- `已完成`

禁止同时叠加多个状态徽标。

## Interaction Rules

### 行点击

- 点击任务主体：设为当前任务
- 不直接开始计时
- 若当前已有其他任务处于运行中的专注态，点击其他任务时提示将切换上下文

### 主按钮

主按钮是唯一主动作，负责开始或恢复专注。

文案规则：

- 普通任务：`开始专注`
- 当前但未运行：`开始当前任务`
- 暂停中：`继续专注`
- 运行中：`专注中`

行为规则：

- 点击普通任务：设为当前任务 + 立即开始
- 点击当前任务：直接开始
- 点击暂停中的当前任务：恢复专注

### 完成按钮

- 点击后标记任务完成
- 若该任务是当前绑定任务，则同步清空任务与计时器绑定
- 若该任务处于运行中专注态，先给予一次轻确认

### 更多菜单

第一版菜单建议包含：

- 编辑任务
- 设为下一项
- 延后到明天
- 移出今日
- 删除任务

若第一版需要收敛范围，最少保留：

- 编辑任务
- 延后到明天
- 移出今日
- 删除任务

## States

### 普通待办

- 展示标题、番茄进度、可选优先级
- 主按钮可点击开始专注

### 当前任务

- 行容器高亮
- 显示 `当前任务`
- 主按钮显示 `开始当前任务`

### 专注中

- 行容器高亮最强
- 显示 `专注中`
- 主按钮进入运行态

### 已完成

- 完成勾选填充
- 标题置灰
- 主按钮隐藏
- 默认不作为今日主列表重点项

### 空状态

空状态应引导用户开始今日计划，而不是简单说明“没有数据”。

- 标题：`今天先安排一件最重要的事`
- 主按钮：`添加任务`
- 次按钮：`从全部任务中选择`

## Sorting Strategy

今日任务默认排序建议：

1. `专注中`
2. `当前任务`
3. `高优先级`
4. `未完成且仍有剩余番茄`
5. `其他任务`

第一版不开放复杂排序设置，以避免过度产品化。

## Data Model Direction

当前 `Task` 已具备：

- `priority`
- `estimatedPomodoros`
- `completedPomodoros`
- `status`

为了支撑真正的“今日任务”，后续建议补充其一：

- `scheduledDate`
- `isPlannedForToday`
- 单独的 `todayTaskIds`

第一版若不扩数据模型，可先沿用现有任务数据，但需要在文案或实现中承认这是“专注任务预览”，不是严格意义的 Today List。

## Timer Integration

### 开始专注

- 若点击的是非当前任务：设为当前 + 绑定计时器 + 开始
- 若点击的是当前任务：直接开始

### 完成一轮番茄后

- `completedPomodoros + 1`
- 列表中的番茄进度实时更新

### 达到预计番茄数后

不强制自动完成任务，而是提供轻确认：

- `已达到预计番茄数，是否标记完成？`

动作：

- `标记完成`
- `继续专注`

### 完成当前任务后

模块应推荐下一项待办：

- `下一项：任务标题`
- 按钮：`开始下一项`

## Component Architecture

建议按职责拆分为：

- `FocusTaskCard`
  - 容器编排
  - 负责数据接线、抽屉、删除确认、头尾摘要

- `FocusTaskSummary`
  - 头部摘要
  - 展示标题、完成数、添加任务按钮

- `FocusTaskList`
  - 负责列表循环、空状态切换、超出数量逻辑

- `FocusTaskListItem`
  - 单条任务项布局与状态表达

- `FocusTaskMeta`
  - 番茄进度与优先级的次信息展示

- `FocusTaskActionsMenu`
  - 更多菜单动作

## Visual Guidelines

- 卡片仍保持现有轻奶油白底和浅边框
- 列表项需要比当前版本更清楚的横向信息分区
- `开始专注` 使用真正按钮，不使用孤立小图标
- 当前任务和专注中任务都应高亮，但层级要区分
- 优先级提示弱于任务标题和番茄进度

## Rollout Priority

按实现优先级排序：

1. 重做 `FocusTaskListItem` 布局和主按钮层级
2. 补 `番茄进度 + 优先级` 次信息展示
3. 增加 `FocusTaskSummary` 头部摘要
4. 扩展 `FocusTaskActionsMenu`
5. 引入真正的“今日任务”数据模型
